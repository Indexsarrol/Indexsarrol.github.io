在Taro小程序中使用原生echarts——ec-canvas

2021-08-22- 2022-06-19
-
Taro > Echarts-ReactCanvas
Indexsarrol

写在前面

在最近的一个小程序版本中，使用了echarts图表，相对传统web端的echarts来说，小程序中嵌入echarts是一件很让人头疼的事，好在echarts官方文档给我提供了已经被封装好的小程序版本的echarts——[ec-canvas](https://echarts.apache.org/zh/tutorial.html#在微信小程序中使用 ECharts)，接下来一起看看如何使用吧！

下载&安装

与传统web端的安装方式不同，小程序版本的echarts是需要自己去其github仓库进行下载，打开之后我们只需要将项目拉下来，然后将整个ec-canvas文件夹复制到项目中去即可。

其中ec-canvas文件夹中包含了以下几个主要文件：

  • ec-canvas.js文件，用于集成echarts中的各种方法；
  • wx-canvas.js文件，封装了WxCanvas类
  • echarts.js文件，核心文件，底层echarts源码。一般该文件都会很大，我们可以使用自定义构建来减少文件的大小。

如何使用

在原生小程序中使用

首先我们需要在pages/line/index.json中添加如下代码，用于引入ec-canvas：

{
  "usingComponents": {
    "ec-canvas": "../../ec-canvas/ec-canvas" // 引入ec-canvas
  }
}

第二步，在index.wxml中使用ec-canvas，其中 ec 是一个我们在 index.js 中定义的对象，它使得图表能够在页面加载后被初始化并设置。

<view class="chart-wrap">
    <ec-canvas id="line-charts" canvas-id="line-charts" ec="{{ ec }}"></ec-canvas>
</view>

第三步，在index.js文件中绘制图表：

function initChart(canvas, width, height) {
    const chart = echarts.init(canvas, null, {
        width,
        height
    });
    canvas.setChart(chart);
 
    const option = {
        // echarts 配置项
    };
    chart.setOption(option);
    return chart;
}
 
Page({
    data: {
        ec: {
            onInit: initChart
        }
    }
});

在Taro小程序中使用

第一步，还是和之前一样，先在index.config.js中引入ec-canvas：

export default {
    navigationBarTitleText: 'Echarts图表',
    onReachBottomDistance: 200,
    enablePullDownRefresh: true,
    usingComponents: {
        // 定义需要引入的第三方组件
        // 1. key 值指定第三方组件名字，以小写开头
        // 2. value 值指定第三方组件 js 文件的相对路径
        'ec-canvas': '../components/ec-canvas/ec-canvas',
    },
};

第二步，在index.jsx 中使用 ec-canvas （简单渲染），其中 ec 是一个我们在 index.jsx 中定义的对象，它使得图表能够在页面加载后被初始化并设置。

import React, { useState } from 'react';
import Taro from '@tarojs/taro';
import { View } from '@tarojs/components';
import * as echarts from '../components/ec-canvas/echarts.js'; // 引入echarts
 
let chart = null;
// 初始化图表
function initChart(canvas, width, height, dpr) {
    chart = echarts.init(canvas, null, {
        width,
        height,
        devicePixelRatio: dpr, // new
    });
    canvas.setChart(chart);
    const option = {
        // ehcarts option配置项
    };
    chart.setOption(option);
    return chart;
}
 
export default function Charts() {
    const [ ec ] = useState({ onInit: initChart });
    return (
        <View className="charts-wrap">
            <ec-canvas id="line" canvas-id="line" ec={ec} />
        </View>
    )
}

第三步，接口请求数据动态渲染图表（常用），我们需要定义一个 chart 对象，用于存放echarts 实例化后的对象，进一步做到获取动态数据。

import React, { useState, useEffect, useCallback } from 'react';
import Taro from '@tarojs/taro';
import { useSelector, useDispatch } from 'react-redux';
import { View } from '@tarojs/components';
import * as echarts from '../components/ec-canvas/echarts.js';
 
export default function Charts() {
    const dispatch = useDispatch();
    const { chartData } = useSelector(store => store.charts); // 从model中获取charts数据
    const [chart, setChart] = useState(null); // 定义chart对象，用于保存echarts实例
    const [ec] = useState({ onInit: initChart }); // 初始化ec对象
     
    // 初始化图表
    const initChart = (canvas, width, height, dpr) => {
        const chart = echarts.init(canvas, null, {
            width,
            height,
            devicePixelRatio: dpr,
        });
 
        canvas.setChart(chart);
        setChartData({});
        setChart(chart);
        return chart;
    }
     
    // 对图表进行塞数据
    const setChartData = useCallback((data) => {
        const option = {
            xAxis: {
                type: 'category',
                data: data?.xAxisData
            },
            yAxis: {
                type: 'value'
            },
            series: [{
                data: data?.data,
                type: 'line'
            }]
        };
        if (chart) chart.setOption(option);
         
    }, [chart]);
 
    // 初始化加载数据并渲染图表
    useEffect(() => {
        if (chart) {
            setChartData(chartData);
        }
    }, [chart, chartData, setChartData]);
 
 
    // 切换数据
    switchData = () => {
        dispatch({
            type: 'xxx/getOilTankCharts',
        });
    }
 
    return (
        <>
            <View onClick={switchData}>切换数据</View>
            <View className="charts-wrap">
                <ec-canvas id="line" canvas-id="line" ec={ec} />
            </View>
        </>
    );
}

以上就是在taro小程序中使用echarts的正确打开方式。但是，新的问题来了，那就是因为echarts的包很大，导致我们小程序就会超出2M的限制。接下来我们介绍分包。

Taro分包

     说到小程序分包，大家应该都不陌生了，具体分包方法点击[这里](https://developers.weixin.qq.com/miniprogram/dev/framework/subpackages/basic.html)查看。在了解分包之前，我还是先说说我遇到的问题吧，场景是这样的，我引入了`echarts`的包，并且也利用分包分了一个油箱模型库的包，当只有一个页面引用`echarts`的时候，`taro`就会将`echarts`分到分到油箱模型库的包内，但是如果有两个及两个页面以上用到了`echarts`，`taro`就会将`echarts`打到主包内，依然导致主包体积超过2M的情况。一开始我以为是我分包的问题，直到我看见了这个：

原来是Taro打包机制的问题。所以只有再去官网找文档了，最终找到了思路，Taro支持自定义 Webpack 配置——webpackChain。大致思路如下：

  1. 利用webpack中的打包机制，将echarts单独打包；
  2. 打包过后在对应页面注入依赖——addChunkPages。

config/index.js 的配置如下：

// ...其他配置
mini: {
    webpackChain(chain, webpack) {
        const config = chain.optimization.get('splitChunks');
        chain.optimization.splitChunks({
            ...config,
            cacheGroups: {
                ...config.cacheGroups,
                [EchartsChunkName]: {
                    name: EchartsChunkName,
                    priority: 50,
                    test: module =>
                        /charts[\\/]components[\\/]ec-canvas[\\/]echarts.js/.test(module.resource),
                },
            },
        });
    },
    addChunkPages(pages) {
        // 相当于注册用到echarts的页面
        pages.set('charts/oilTankAdd/index/index', [EchartsChunkName]);
        pages.set('charts/oilTankDetail/index/index', [EchartsChunkName]);
        pages.set('charts/oilTankEdit/index/index', [EchartsChunkName]);
    },
    commonChunks(commonChunks) {
        return commonChunks;
    },
},

如果按照上面的代码其实就已经完成，但是会出现一个让人很恶心的问题，那就是ec-canvas组件找不到echarts模块依赖。如下图所示，图片摘自网络

所以这个时候 就需要我们干一件大事——改源码！！！需要我们手动去引用echarts，但是这种就需要我们每次打包过后都要去手动引用，就特别的那啥！所以我们需要再添加一个配置，让他自动引用——webpack.BannerPlugin，具体配置如下：

webpackChain(chain, webpack) {
    const config = chain.optimization.get('splitChunks');
    chain.optimization.splitChunks({
        ...config,
        cacheGroups: {
            ...config.cacheGroups,
            [EchartsChunkName]: {
                name: EchartsChunkName,
                priority: 50,
                test: module =>
                        /charts[\\/]components[\\/]ec-canvas[\\/]echarts.js/.test(module.resource),
            },
        },
    });
    // 添加如下配置即可
    chain.plugin('banner').use(
        new webpack.BannerPlugin({
            banner: "require('../../echarts');",
            raw: true,
            include: /charts[\\/]components[\\/]ec-canvas[\\/]ec-canvas.js/, // 这里写你自己项目对应的文件
        }),
        [],
    );
},

这样就大功告成了！
