    写在前面

    今天的分享主要是围绕ts + hooks在项目中的使用展开的，那么我们为什么要使用ts呢，第一，区别于js，ts有着很强大的类型约束，使得我们在日常开发中可以得心应手的处理一些因类型错误带来的不可预估的问题；第二，ts在编辑器中的优势，相信大家在码代码的时候肯定都用过代码提示，那么ts其实就提供了这样一种很方便的工具。举个例子，当我们使用Button组件时，我们希望设置type属性，我们只需要输入type后，vscode中的代码提示就会自动的将type的枚举带出来。接下来，我们稍微回顾一下TS 和 React-Hooks吧！

    TypeScript回顾

    由于只是简单回顾，很多细节性的知识点就不说了，我们就稍微回顾一下我们即将会用到的一些涉及到ts的东西。

    type声明类型

    如果我们希望声明一个含有多种type的类型，我们就可以使用type进行声明，例如：

    type season = 'spring' | 'summer' | 'autumn' | 'winter';
type TPerson = {
	name: string,
	age: number,
	sex: string
};

const season: season = 'spring';
const person: TPerson = {
	name: '章三',
	age: 18,
	sex: '男'
};

    interface接口声明

    interface IProps {
	name: string,
	age: number,
	sex: string
};
const person: TPerson = {
	name: '章三',
	age: 18,
	sex: '男'
};

    工具类型Partial、Omit<K,T>

    Partial

    Partial 可以快速把某个接口类型中定义的属性变成可选的(Optional)，举个例子：

    interface People {
  age: number;
  name: string;
}

const Jerry:People = {
    age: 10,
    name: 'Jerry'
};

const Tom: People = { // 会报错
    name: 'Tom'
}

// 解决
type AnonymousPeople = Partial<People>;

const tom:AnonymousPeople = {
    name: 'Tom'
};

    Omit<K,T>

    Omit<K,T>类型让我们可以从另一个对象类型中剔除某些属性，并创建一个新的对象类型：
    K：是对象类型名称，T：是剔除K类型中的属性名称，例如：

    interface IUserProps {
	name?: string,
	age?: number,
	sex?: string
}
// 如果我即将需要声明的对象不需要年龄，就可以这么写
type newUserProps = Omit<IUserProps, 'age'>
// 相当于
newUserProps {
	name?: string,
	sex?: string
}

    React中常用类型声明

    React给我们提供了日常开发中，所需要的各种类型，其中比较常用的有React.FC、React.ReactNode、React.CSSProperties。

    声明函数式组件

    const Button: React.FC<T> = () => {}

    声明React节点

    const Child: React.ReactNode = (props) => {
	return props.children;
}

    声明行内样式类型

    const style: React.CSSProperties = { color: '#fff', background: '#fff' };

    React-Hooks回顾

    由于只是简单回顾，很多细节性的知识点就不说了，我们就稍微回顾一下我们即将会用到的一些涉及到hooks的一些方法。

    useState()

    useState()方法就是类似于类组件中的setState()方法，只不过useState函数返回一个数组，当前 state 以及更新 state 的函数，同时接受一个默认参数。

    import React, { useState } from 'react';
const [count, setCount] = useState(0);

    useEffect()

    useEffect()函数接受两个参数，第一个参数是一个回调函数，第二个参数是一个数组依赖项。形式如下：

    import { useEffect } from 'react';
useEffect(() => {
	console.log(1234);
}, [])

    其中，最主要的就是第二个参数——依赖项数组。

    如果为空数组时，就相当于类组件中的钩子函数——componentDidMount(),只执行一次。

    如果为非空数组时，则相当于类组件中的钩子函数——componentDidUpdate()，仅当数组的依赖项发生变化时执行回调函数。

    这里就简单的介绍一下这两个hook的简单使用方式。

    搭建开发环境

    为了简单起见，我们就不去自己手动的搭建脚手架了，直接使用create-react-app的TS版本即可。我们直接按照官网的安装方式下载安装即可，这里要注意一点就是一定要下载ts的版本！！！安装成功后，即可使用，下面是其目录结构，因为我们只是简单的封装组件作为演示，所以就没有必要去规整机构目录，直接沿用他的即可：

    封装简单组件——Button组件

    因为仅仅是演示，我们所有的样式文件就不写了，我们只写业务逻辑即可。首先我们新建Button/Button.tsx文件，导入常用的工具模块，其中classnames是一个动态类名拼装函数，具体使用查看这里:

    import React from 'react';
import classnames from 'classnames';
import styles from './index.scss';

    接下来，我们声明各种类型：

    export type ButtonType = 'default' | 'primary' | 'link' | 'dashed' | 'danger' | 'warning'; // 按钮类型
export type ButtonSize = 'large' | 'small'; // 按钮尺寸
export interface IButtonProps {
    className?: string; // 类名
    btnType?: ButtonType; // 类型
    size?: ButtonSize; // 设置按钮尺寸，可选值为 `large`， `small`，默认值为`default`。
    disabled?: boolean; // 按钮失效状态
    href?: string; // 点击跳转的地址，指定此属性 button 的行为和 a 链接一致，只在btnType为`link`时生效。
    icon?: IconProp; // 图标
    loading?: boolean; // 加载中
    children?: React.ReactNode
}

type NativeButtonProps = IButtonProps & React.ButtonHTMLAttributes<HTMLElement>;
type AnchorButtonProps = IButtonProps & React.AnchorHTMLAttributes<HTMLElement>;
export type ButtonProps = Partial<NativeButtonProps & AnchorButtonProps>;

    最后，我们正式封装组件：

    const Button: React.FC<ButtonProps> = (props) => {
    const {
        btnType,
        className,
        size,
        disabled,
        href,
        children,
        loading,
        icon,
        ...restProps
    } = props;
    const prefix = 'idx-btn';
    const classNames = classnames(prefix, className, {
        [`${prefix}-${btnType}`] : btnType,
        [`${prefix}-${size}`] : size,
        disabled: btnType === 'link' && disabled
    });
    if (btnType === 'link' && href) {
        return (
            <a
                className={classNames}
                href={href}
                {...restProps}
            />
                {children}
            </a>
        );
    } else {
        return (
            <button {...restProps} className={classNames} disabled={loading || disabled}>
                <div className={`${prefix}-wrapper`}>
                    { loading && <Icon icon="spinner" spin style={{ marginRight: 6 }}/> }
                    <span>{ icon && <Icon icon={icon} style={{ marginRight: 6 }} /> }</span>
                    <span>{children}</span>
                </div>
            </button>
        );
    }
}

Button.defaultProps = {
    btnType: 'default',
    disabled: false
};

    这样我们的Button组件就封装完成了，那么至此其实我们并没有看出来Button组件使用了TS之后的差异，别着急，我们往下看。我们怎么去使用这个组件呢，举个例子：

    import Button from './Button';
 
function Example() {
    return (
        <Button btnType="primary">按钮</Button>
    );
}

    因为现在我们在ts中定义的接口中，size属性是可选的，所以我们看不出来，此时我们变更size属性为必选，然后我们故意不传递size属性会发生什么呢？

    export interface IButtonProps {
    className?: string; // 类名
    btnType?: ButtonType; // 类型
    size: ButtonSize; // 设置按钮尺寸，可选值为 `large`， `small`，默认值为`default`。 更改为必填属性
    disabled?: boolean; // 按钮失效状态
    href?: string; // 点击跳转的地址，指定此属性 button 的行为和 a 链接一致，只在btnType为`link`时生效。
    icon?: IconProp; // 图标
    loading?: boolean; // 加载中
    children?: React.ReactNode
}

    我们在编辑器中就会看到报错，鼠标移上去就会显示这么一串东西：

    这就很明显的说明了，size属性是必传的，我们只要添加size属性之后，这个报错就消失了。

    封装复杂组件——Tabs组件

    接下来，我们稍微的增加点难度，封装一个稍微复杂的组件——Tabs组件，这个组件可以释放出更多的页面空间，来展示更多的东西。首先我们来看一下已经封装好的组件是如何使用的：

    import { Tabs } from 'xxx';
import ReactDOM from 'react-dom';
 
ReactDOM.render(
    <Tabs>
        <Tabs.TabPane>标题1</Tabs.TabPane>
        <Tabs.TabPane>标题2</Tabs.TabPane>
    </Tabs>
    ,
    mountNode
);

    我们可以看出来，其实一个Tabs组件包括了外层Tabs以及内层元素TabPane组件，所以我们要封装两层，从内而外开始，接下来开始吧！

    TabPane组件

    首先新建TabPane.tsx组件，导入常用模块：

    import * as React from 'react';
import classnames from 'classnames';

    接着，我们声明一下常用的类型：

    export interface ITabPaneProps {
  key: string;
  tab: string | React.ReactNode;
  className?: string;
  disabled?: boolean;
  style?: React.CSSProperties
}

    最后封装TabPane组件：

    const TabPane: React.FC<ITabPaneProps> = (props) => {
  const { className, style, children } = props;
  const panelClasses = classnames('idx-tabs-panel', className)
  return (
    <div className={panelClasses} style={style}>
      {children}
    </div>
  );
}
 
TabPane.displayName = 'TabPane'; // 定义名称，便于Tabs组件判断传入的组件是否为TabPane

    Tabs组件

    首先新建Tabs.tsx组件，导入常用模块：

    import React, { useState } from 'react';
import classnames from 'classnames';
import styles from './index.scss';

    接着，我们声明一下常用的类型：

    export type TabsType = 'line' | 'card'; // 定义Tabs的类型，line为线型，card为卡片状
 
export type TabPosition = 'top' | 'right' | 'bottom' | 'left'; // 定义头部在哪个位置
 
export interface ITabsProps { // 定义Tabs需要传递的参数
  defaultKey: string; // 默认key值，用于展示选中项
  className?: string; // 类名
  style?: React.CSSProperties; // 额外css样式
  type?: TabsType; // 理性
  tabPosition?: TabPosition; // 位置
  tabBarExtraContent?: React.ReactNode; // 额外内容
  onChange: (key: string) => void // 点击切换触发的回调函数
}

    最后我们开始封装组件，这里要注意一下，因为Tabs组件是分为头部切换区域以及内容区域，所以我们会定义两个方法，分别用于渲染头部和内容区域，且TabPane组件是放在Tabs组件内部，具体如下：

    const Tabs: React.FC<ITabsProps> = (props) => {
  const {
    defaultKey,
    className,
    tabPosition,
    style,
    children,
    onChange,
    type,
    tabBarExtraContent
  } = props;
  const [ currentKey, setCurrentKey ] = useState(defaultKey);
 
  // 切换事件
  const handleChange = (key: string) => {
    setCurrentKey(key);
    onChange(key)
  }
 
  const tabsClasses = classnames('idx-tabs', className, {
    [`idx-tabs-${tabPosition}`]: tabPosition,
    [`idx-tabs-${type}`]: type
  });
   
  const tabsNavClasses = classnames('idx-tabs-bar', {
    [`idx-tabs-${tabPosition}-bar`]: tabPosition
  });
     
  // 渲染头部
  const renderHeader = () => {
    return React.Children.map(children, (child, index) => { // 遍历children
      const childElement = child as React.FunctionComponentElement<ITabPaneProps>; // 类型断言
      const { tab, disabled } = childElement.props;
      const tabsNavItemClasses = classnames('idx-tabs-tab', { // 添加选中样式
        'idx-tabs-tab-active': currentKey === index.toString(),
        'idx-tabs-tab-disabled': disabled
      });
      return (
        <div
          className={tabsNavItemClasses}
          onClick={disabled ? () => {} : () => {handleChange(index.toString())}}
        >
          {tab}
        </div>
      );
    })
  }
    
  // 渲染内容区域
  const renderChildren = () => {
    return React.Children.map(children, (child, index) => { // 遍历children
      const childElement = child as React.FunctionComponentElement<ITabPaneProps>; // 类型断言
      if (childElement.type.displayName === 'TabPane') { // 判断是否为TabPane组件
        if (index.toString() === currentKey) {
          return childElement;
        }
      } else {
        console.error('Warning: Tabs component only accept component! ')
      }
    })
  }
   
  return (
    <div className={tabsClasses} style={style}>
      <div className={tabsNavClasses}>
        {
          tabPosition === "top" || tabPosition === "bottom"
          ?
          <>
            <div className="idx-tabs-bar-container">
            {renderHeader()}
            </div>
            <div className="idx-tabs-tab-bar-extra">
              {tabBarExtraContent}
            </div>
          </>
          :
          renderHeader()
        }
      </div>
      <div className="idx-tabs-content">
        {renderChildren()}
      </div>
    </div>
  );
}
 
Tabs.defaultProps = {
  defaultKey: '0',
  type: 'line',
  tabPosition: 'top'
}
